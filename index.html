📖 Les fondamentaux de Kubernetes

            
Kubernetes (K8s) est un orchestrateur de conteneurs open-source. Il automatise le déploiement, la mise à l’échelle et la gestion des applications conteneurisées.


            
🧩 Concepts clés (avec exemples commentés)

            

                
Concept Pod
Unité la plus petite déployable. Contient un ou plusieurs conteneurs partageant le réseau.
apiVersion: v1
kind: Pod
metadata:
  name: mon-pod
  labels:
    app: nginx
spec:
  containers:
  - name: nginx
    image: nginx:latest
    ports:
    - containerPort: 80

                
Concept Service
Expose un ensemble de Pods via une IP stable. Types : ClusterIP, NodePort, LoadBalancer.
apiVersion: v1
kind: Service
metadata:
  name: mon-service
spec:
  selector:
    app: nginx
  ports:
  - protocol: TCP
    port: 80
    targetPort: 80
  type: ClusterIP

                
Concept Ingress
Routeur HTTP/HTTPS externe. Redirige vers les services selon le nom d'hôte ou le chemin.
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: mon-ingress
spec:
  rules:
  - host: monapp.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: mon-service
            port:
              number: 80

                
Concept StorageClass
Définit un provisionneur de volumes persistants (ex: local-path, EBS).
apiVersion: storage.k8s.io/v1
kind: StorageClass
metadata:
  name: local-path
provisioner: rancher.io/local-path
reclaimPolicy: Delete

                
Concept ConfigMap
Stocke des données de configuration non sensibles (variables d'environnement).
apiVersion: v1
kind: ConfigMap
metadata:
  name: app-config
data:
  APP_COLOR: blue
  APP_MODE: production

                
Concept Deployment
Gère un ensemble de Pods avec rolling update, rollback et scaling.
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deploy
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.21

            


            
🛠️ Commandes kubectl – Explications & Exemples

            
🔍 kubectl cluster-info
Affiche l'URL du serveur API.
$ kubectl cluster-info
Kubernetes control plane is running at https://172.18.0.2:6443
💡 Vérifie que vous êtes bien connecté au cluster.

            
📋 kubectl get nodes
Liste les nœuds avec leur statut.
$ kubectl get nodes
NAME                 STATUS   ROLES           AGE
kind-control-plane   Ready    control-plane   5m
kind-worker          Ready    <none>         5m
💡 Utilisez -o wide pour plus de détails (IP, version).

            
🔎 kubectl describe pod <nom>
Détails et événements d'un Pod (très utile pour le debug).
$ kubectl describe pod mon-pod
...
Events:
  Type    Reason     Age   Message
  Normal  Scheduled  2m    Successfully assigned...
  Normal  Pulling    2m    Pulling image "nginx"

            
📄 kubectl apply -f fichier.yml
Crée ou met à jour des ressources.
$ kubectl apply -f deployment.yml
deployment.apps/nginx-deploy created

            
📜 kubectl logs <pod>
Affiche les logs du conteneur.
$ kubectl logs nginx-deploy-7d8f8-5xg7f
127.0.0.1 - - [10/Jan/2024] "GET / HTTP/1.1" 200

            
🖥️ kubectl exec -it <pod> -- /bin/bash
Ouvre un shell interactif dans le conteneur.
$ kubectl exec -it mon-pod -- /bin/bash
root@mon-pod:/# ls


            
🔒 Sécurité & Administration Infrastructure

            
Objectif : Garantir la confidentialité, l'intégrité et la disponibilité du cluster.

            
            
1. RBAC (Contrôle des accès)

            
Définit qui peut faire quoi sur quelles ressources.

            
# Role : lecture des Pods dans le namespace "dev"
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  namespace: dev
  name: pod-reader
rules:
- apiGroups: [""]
  resources: ["pods"]
  verbs: ["get", "list"]
---
# RoleBinding : lie le Role à l'utilisateur "alice"
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata:
  namespace: dev
  name: alice-binding
subjects:
- kind: User
  name: alice
roleRef:
  kind: Role
  name: pod-reader
💡 Vérification : kubectl auth can-i list pods --as=alice -n dev


            
2. Network Policies (Isolation réseau)

            
Par défaut, tous les Pods communiquent. Les Network Policies filtrent le trafic.

            
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: backend-policy
spec:
  podSelector:
    matchLabels:
      app: backend
  policyTypes:
  - Ingress
  ingress:
  - from:
    - podSelector:
        matchLabels:
          app: frontend
    ports:
    - protocol: TCP
      port: 8080
💡 Nécessite un CNI supportant les Network Policies (Calico, Cilium, Weave).


            
3. Pod Security Standards (PSS)

            
Profils : Privileged, Baseline, Restricted. Appliqués via les labels de namespace.

            
apiVersion: v1
kind: Namespace
metadata:
  name: production
  labels:
    pod-security.kubernetes.io/enforce: restricted


            
4. Gestion sécurisée des Secrets

            
Les Secrets sont encodés en base64, mais non chiffrés par défaut dans etcd. Activez le chiffrement au repos.

            
# EncryptionConfiguration pour chiffrer les Secrets
apiVersion: apiserver.config.k8s.io/v1
kind: EncryptionConfiguration
resources:
- resources:
  - secrets
  providers:
  - aescbc:
      keys:
      - name: key1
        secret: <base64-key>
  - identity: {}


            
5. Resource Quotas & Limit Ranges

            
Protègent le cluster contre les dénis de service.

            
apiVersion: v1
kind: ResourceQuota
metadata:
  name: compute-quota
  namespace: dev
spec:
  hard:
    requests.cpu: "4"
    requests.memory: "8Gi"
    pods: "10"


            
6. Security Context (Moindre privilège)

            
apiVersion: v1
kind: Pod
metadata:
  name: secure-pod
spec:
  securityContext:
    runAsNonRoot: true
    runAsUser: 1000
  containers:
  - name: app
    image: nginx
    securityContext:
      allowPrivilegeEscalation: false
      capabilities:
        drop: ["ALL"]
      readOnlyRootFilesystem: true


            
7. Audit Logs

            
Enregistre toutes les requêtes vers l'API Server pour la conformité.

            
# Politique d'audit (exemple)
apiVersion: audit.k8s.io/v1
kind: Policy
rules:
- level: Metadata
# Démarrer l'API Server avec :
--audit-policy-file=/etc/kubernetes/audit-policy.yaml
--audit-log-path=/var/log/kubernetes/audit.log
